# Source Query Workflow

## Rule

在需要查询项目源码或 Mindustry 源码前，先查询 `knowledge/` 知识库。

如果知识库内容满足下面任一情况，再进入源码查询：

- 没有覆盖当前问题
- 信息模糊、不够具体
- 与当前代码状态不一致

完成源码查询并得到更准确结论后，要回写或补充 `knowledge/`，保证后续同类问题优先通过知识库解决。

## Execution Order

1. 先查 `knowledge/index.md` 和相关知识条目。
2. 如果知识库可直接回答，就基于知识库继续工作。
3. 如果知识库不足，再查项目源码或外部源码。
4. 得出明确结论后，更新对应知识条目或新增条目。
5. 在 `knowledge/log.md` 记录这次知识补充。

## Notes

- 知识库优先，源码兜底。
- 源码查询不是替代知识库，而是用于澄清和补全知识库。
- 新增知识时优先写成可复用结论，而不是一次性问题记录。
